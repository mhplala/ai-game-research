# 模块 06A — 差异化产品机会（产品经理 / 创业者视角）

> 上游主命题：**Runtime AI × 小游戏是真空带（Token 经济学卡死），但 Generative Playable Feed(GPF）子品类正在用 Pipeline AI + UGC 飞轮打破它 — 窗口期 12-24 个月。**

本文件回答一个问题：**"我想立项一款 AI × 小游戏产品，应该做什么？"**

---

## §0. TL;DR（给 PM / 创业者的 30 秒版本）

**一句话：现在应该做 Generative Playable Feed 的中国版（P1），或者做端侧小模型的 AI-native 小游戏（P2),12-24 个月是黄金窗口期。**

三个核心判断：

1. **不要碰 Runtime AI × IAA 小游戏**（纯开放对话式 NPC 驱动玩法）。Token 经济学在 ARPDAU ≤ 0.3 元的环境下卡死（PROGRESS §3.3），每个 DAU 每日 AI 成本 0.1-0.5 元就能把毛利吃光。除非你有豆包内部 Token 补贴，否则 12 个月内不要做。
2. **应该做 Pipeline AI**（生成阶段用 AI，运行时 0 Token)+ **UGC 飞轮**（玩家 prompt → 玩家 remix → 病毒传播）。Rezona、LoopIt、YouTube Playables Builder 已经跑通北美 / 欧洲，中文市场玩家数约等于 0（玩家全景图 1.4 的 Pipeline-Creator 象限空白）。
3. **或者绕开"游戏"品类本身**，做"游戏壳 ≠ 游戏"的陪伴 / 教育 / 工具产品（P3），躲开版号 + 把游戏留存机制嫁接到订阅 ARPU 3-5x 的赛道。

**最值得投入的单一机会：P1 中文版 GPF**，理由在 §3 展开。

---

## §1. 产品机会总清单（8 张机会卡）

---

### ⭐ P1. 中文版 Generative Playable Feed(GPF）—— 最强推荐

**一句话 SKU 描述**：一个中文 text-to-mini-game 移动 App / 小程序，用户输入 prompt（或选 template)→ 30 秒内生成一个 30 秒可玩的 mini playable → 信息流呈现 → 支持 remix + fork + 分享。前 6 个月只做 5 种玩法 template（跑酷 / 消除 / 投射 / 躲避 / 节奏），与豆包 API 深度绑定。对标 Rezona（美国）+ LoopIt(MWM.ai，法国）+ YouTube Playables Builder。

#### Market Pull

- **目标用户画像**:14-24 岁中国城市 Z 世代，TikTok / 小红书 / B 站 重度用户，会用"捏人 / 捏角色 / 捏表情包"类 UGC 工具，不会写代码但喜欢"创造"的自我表达。女性略多（对标剪映 / 醒图 / 无限暖暖）。
- **痛点**：目前中文世界里，非程序员想做一个游戏 0 路径（Roblox 中国版 罗布乐思 已关停，Crayta / Core / Rec Room 无中文版），Godot / Unity 学习曲线劝退 99%。
- **当前替代方案**：剪映（只做视频不做交互）+ 微信小游戏（只做消费不做创作）+ 无代码工具（爱贝 / 口袋 42 已死）。市场上**没有**"30 秒生成可玩游戏 + 信息流"的中文产品。
- **市场空白证据**：玩家全景图 1.4 中 Pipeline-Creator × 中文区象限：玩家数 = 0。Rezona 2025 Q3 数据：DAU 40 万，70% 内容由 UGC 产生，次日留存 35%(PROGRESS §2.5）。
- **TAM 估算**：若对标剪映创作者端 1.5 亿 MAU × 5% 愿意尝试交互创作 = 750 万 MAU ceiling；保守首年目标 50 万 MAU。

#### Tech Push

- **核心技术栈**：豆包 / DeepSeek / Qwen 作为 LLM 生成层（负责把自然语言 → JSON 游戏参数 + 对白 + 关卡描述）；**Phaser / Cocos / LayaBox** 作为运行时引擎（0 Token，完全端侧执行）；Stable Diffusion / 可灵 作为美术资产生成层（异步，缓存复用）。
- **供应链现状**:2025 年豆包 1.5-pro 输入 0.8 元/百万 Token，生成一个 mini game ≈ 2000 Token ≈ 0.0016 元，成本可控。Cocos 小游戏引擎中文生态成熟，微信小游戏 / 抖音小游戏双端 ready。
- **技术可行性状态**:**已被验证**。Rezona、LoopIt、YouTube Playables Builder 三个北美 / 欧洲产品已跑通 generation-time AI + runtime-free 架构（PROGRESS §2.5）。中文版几乎是工程复刻，难点在 template 设计和 prompt 工程，不在技术。
- **开发门槛**:0 → MVP 约 4 个月，5 人团队（1 PM + 2 游戏工程师 + 1 AI 工程师 + 1 设计师），预算 150-200 万。

#### 中国特殊性

- **优势**:(a) 豆包 Token 补贴可直接把 generation cost 打到 0;(b) 抖音小游戏 / 微信小游戏分发渠道比北美 App Store 高效 10x;(c) 中文 prompt → 中文游戏资产的链路独家，海外产品因为多语言复杂度难以本地化；（d) 剪映 / 醒图 已经教育过"AI 生成 + 一键分享"的 UGC 心智。
- **障碍**:(a) **版号**：若定位为"游戏"需要版号，路径漫长；解法是**定位为"互动内容创作工具"**（参考剪映 / 美图），跑审核而非版号；（b) 内容审核：UGC 生成需要 LLM 侧 + 内容安全双审，这是刚性成本；（c) 未成年人保护：需严格限制发布权限。
- **豆包 Token 补贴路径**：字节生态内可做官方 App,Token 内部结算；外部创业可申请火山引擎初创扶持（已知 2025 Q4 有 AI 应用 200 万 Token 免费额度）。

#### 可执行性清单

- **Week 1-4**：锁定 5 个玩法 template（跑酷 / 消除 / 投射 / 躲避 / 节奏），做 prompt-to-JSON schema,1 个可跑的端到端 demo（用户输入"猫猫躲避西瓜" → 出可玩页面）。
- **Month 2-3**：内测版本，100 个种子用户（小红书 / B 站 招募），测 generation 成功率（目标 > 80%）、可玩性（目标 > 70% 的生成作品可被他人玩 ≥ 30 秒）。
- **Month 4-6**：公测，嵌入抖音小游戏 / 微信小游戏，接入豆包 API，发起"#我的 AI 小游戏"挑战赛，目标 50 万 DAU 注册、5 万 DAU 留存。
- **Month 6-12**：信息流推荐算法上线（复用抖音推荐逻辑），UGC 飞轮正反馈，Remix 率目标 > 30%，次日留存 > 25%。

#### 风险与规避

- **风险 1**：内容审核成本爆表（UGC + LLM 双重）。**规避**：前 6 个月只开放 template 内选择，不做完全自由 prompt；接入网易易盾 / 阿里绿网。
- **风险 2**：生成质量不稳定，80% 的生成作品"不好玩"。**规避**：引入"社区编辑推荐"+ AI 质量打分过滤，只 surface top 20%。
- **风险 3**：字节 / 腾讯 官方复刻。**规避**：抢 12 个月时间窗，拿到早期创作者社群 + prompt 数据 moat。

#### 成功标准（12 个月 KPI)

- 50 万累计注册用户 / 5 万 DAU / 次日留存 > 25%
- UGC 作品累计 100 万个，Remix 率 > 30%
- 单 DAU generation cost < 0.05 元（含 Token + 审核）
- 完成 A 轮融资 或 进入字节战投雷达

### P2. 端侧小模型 AI-native 小游戏

**一句话 SKU 描述**：做一款 WebGPU + Llama 3.2-3B（或 Qwen2.5-3B 量化版）端侧推理的 AI-native 小游戏，核心玩法是"玩家用自然语言说服 / 辩论 / 社交欺诈"NPC,Token 成本 = 0（完全跑在用户设备上）。对标 Suck Up!(Steam 付费 $15)+ AI People，但做成**免费小游戏 + IAA** 形态。

#### Market Pull

- 目标用户：16-30 岁男性为主，喜欢狼人杀 / 剧本杀 / AI Dungeon / 沙雕短视频，有至少 8GB RAM 的 iPhone 14 / 骁龙 8 Gen 2+ 安卓机。
- 痛点：（a) Suck Up! 要付费 + 需要 PC;(b) Character.AI 不是游戏只是聊天；（c) 中文 AI 游戏几乎只有对话 Demo 没有真玩法。
- 当前替代方案：彩云小梦（文字冒险，非游戏）、剧本杀 App（无 AI，需要真人）。
- 市场空白证据：PROGRESS §3.4 识别的"移动端 on-device LLM × game"象限几乎无产品，WebGPU 在 iOS 17 / Chrome / 微信 WebView 已经就绪。
- TAM 估算：中国 AI 原生尝鲜用户 2000 万，游戏子集 500 万，目标首年 30 万 DAU。

#### Tech Push

- 核心栈：Llama.cpp / MLC-LLM 量化模型（3B Q4,1.5GB 内存占用）+ Unity / Cocos 游戏层 + WebGPU 推理后端。
- 技术可行性状态：**边缘可行**。2025 Q3 MLC-LLM 在 iPhone 15 Pro 上 3B 模型可达 15 tok/s，骁龙 8 Gen 3 约 10 tok/s。仍有 10% 低端机型兼容性问题。
- 开发门槛：0 → MVP 约 6 个月，6 人团队，预算 250-300 万（端侧推理是硬技术）。

#### 中国特殊性

- 优势：中文量化小模型（Qwen2.5-3B、MiniCPM-4B、InternLM2-1.8B）全球最强，且全部开源免费商用。
- 障碍：（a) 版号；（b) 端侧模型热更新体验差；（c) 低端机型占比高（小米 Redmi 系列）降低覆盖率。

#### 可执行性清单

- **Week 1-4**：技术验证，Qwen2.5-3B 在 iPhone 13 / 红米 Note 13 Pro 上跑通，测 tok/s 基线。
- **Month 2-3**：玩法设计，定一个核心玩法 —— 推荐"AI 狼人杀单机版"(4 个 AI NPC + 玩家）。
- **Month 4-6**:MVP Demo 发布内测，目标 500 次完整对局。
- **Month 6-12**:TapTap / 抖音 小游戏上线（先不走版号，作为 Demo / Beta），目标 30 万 DAU。

#### 风险与规避

- **风险 1**:3B 模型 RP 能力差，玩家对话体验翻车。**规避**：大量 fine-tune + 角色 prompt 工程 + 预设对话树兜底。
- **风险 2**：机型覆盖率低。**规避**：分级降级 — 高端机跑 3B，中端跑 1.8B，低端走云端（对少数付费用户开放）。

#### 成功标准（12 个月 KPI)

- 30 万 DAU，留存次日 > 20%、7 日 > 10%
- 单局游戏时长 > 10 分钟
- 0 Token 云成本（除低端兜底）

### P3. "游戏壳 ≠ 游戏" 陪伴 / 教育 / 工具产品

**一句话 SKU 描述**：做一款不以"游戏"为名的产品，但把游戏的核心留存机制（目标梯度 + 变动奖励 + 社交比较）嫁接进去。优先选三个子方向：（a)**AI 陪伴 × 养成游戏**（对标星野 / 猫箱 + AI 角色；养成 + 好感度系统是游戏核）；（b)**AI 英语 × Duolingo Max 中文版**（对标多邻国 AI Roleplay，但做成中文英语学习）；（c)**AI 心理陪伴 × 情感日记**（对标 Replika + Finch）。

#### Market Pull

- 目标用户：（a) 陪伴 → 18-26 女性，原乙女 / 泛 ACGN 用户；（b) 教育 → 20-35 岁都市白领学英语 / 雅思；（c) 心理 → 22-32 岁高焦虑城市白领。
- 痛点：（a) 星野 / 猫箱 已验证 PMF 但留存仍是"对话型 APP 天花板 3 个月退坑"(PROGRESS §4.4);(b) 多邻国 Max 无中文版；（c) 国内心理 App 同质化严重。
- 市场空白证据：订阅 ARPU 是 IAA 小游戏 3-5x(PROGRESS §1.4），且绕开版号。
- TAM：陪伴赛道 MAU 已突破 5000 万（星野 + 猫箱 + 筑梦岛）；教育英语赛道 2000 万付费用户。

#### Tech Push

- 核心：大模型对话（豆包 / 通义 / DeepSeek)+ 轻量游戏引擎（UI 动画为主）+ 订阅系统。Token 成本高但订阅模型能覆盖。
- 技术可行性：完全可行，已有星野 / 猫箱 先例。
- 开发门槛：3-6 个月 MVP,8 人团队，预算 200-400 万。

#### 中国特殊性

- 优势：（a) **绕开版号**（注册为"社交娱乐"或"教育"类目）；（b) 豆包 Token 补贴；（c) 订阅用户付费习惯已培养完成。
- 障碍：（a) 陪伴赛道已红海，需要**很强的差异化角**（比如：AI NPC 有主动行为 + 可养成家族树）；（b) AI 备案要求。

#### 可执行性清单

- **Week 1-4**：锁定一个垂类 — 推荐 AI 养成陪伴（竞品最激烈但也最大）。
- **Month 2-3**:MVP，核心玩法"AI 角色养成树 + 好感度 + 日常任务"。
- **Month 4-6**：小红书 / B 站种草 + TapTap 上线。
- **Month 6-12**：订阅体系完善，目标月付费用户 5 万、ARPU 30 元。

#### 风险与规避

- 风险：陪伴赛道竞争极激烈，头部效应强。**规避**：深耕小众细分（如"AI 猫咪养成 + 宠物社区"）。

#### 成功标准

- 月付费用户 5 万，MRR 150 万，次月留存 > 40%

### P4. AI NPC 中间件 / SDK for Chinese Devs

**一句话 SKU 描述**：做 Inworld AI 的中国版 —— 一个 SaaS / SDK，让中国游戏开发者一键接入"人设 + 记忆 + 语音 + 立绘 + 内容安全审核"全套 AI NPC 能力。对标 Inworld AI（美国 $500M 估值），但中文优先、开源核心、按 MAU 订阅。

#### Market Pull

- 目标用户：B 端 — 中国中小游戏开发团队（Cocos / Unity 开发者约 100 万、TapTap 独立游戏开发者约 10 万）、二次元 / 乙女 / 剧情类手游 CP。
- 痛点：G01 Token 成本、G02 角色一致性、G04 内容安全，每家自己做都要 3 个月 + 100 万，重复造轮子。
- 市场空白：中国没有 Inworld 的对标，游戏 AI 中间件只有网易伏羲（不对外）和米哈游自研（不对外）。
- TAM:B 端 SaaS，单客单价 5 万/年，目标 200 个付费客户 = 1000 万 ARR。

#### Tech Push

- 核心：豆包 / DeepSeek API + 内存/向量库（Milvus)+ TTS（火山引擎）+ 立绘生成（SD 微调）+ 内容安全中间层。
- 技术可行性：成分拼装可行，关键是中文内容安全 + 记忆长上下文。
- 开发门槛：6-9 个月，10 人团队，预算 400-600 万。

#### 中国特殊性

- 优势：中文 LLM + 中文 TTS + 中文内容安全，海外 Inworld 难以迁移。
- 障碍：游戏 B 端预算紧缩，付费意愿低于美国；需要"杀手级 demo 游戏"证明 ROI。

#### 可执行性清单

- **Week 1-4**:MVP SDK,Cocos + Unity 双版本，提供 3 个预设人设 demo。
- **Month 2-3**:2-3 个种子客户免费试用（偏向乙女 / 文字游戏）。
- **Month 4-6**：完善记忆 + 语音 + 立绘三合一。
- **Month 6-12**:PMF 验证，目标 20 家付费客户。

#### 风险与规避

- 风险：网易 / 腾讯 / 米哈游 下场做同样的东西，打白牌。**规避**：做开源核心建开发者生态。

#### 成功标准

- 20 家付费 + 500 家免费接入，ARR 100 万

### P8. AAA Runtime AI 合作 / 工具 / License 模式

**一句话 SKU 描述**:**不自己做游戏**，而是把 Runtime AI 能力打包卖给有钱的 AAA / 中型 CP。对标 Krafton MOGIA × Inworld / Ubisoft NEO NPC。

#### Market Pull

- 目标客户：中国 Top 20 CP（米哈游 / 网易 / 腾讯 / 莉莉丝 / 叠纸 / 鹰角 / 网易大世界团队）。
- 市场空白：国内 AI × AAA 合作几乎 0 案例，只停留在 Tech Demo。

#### Tech Push / 可行性

- 需要深度技术 + 行业关系，门槛高，但 ASP 极高（单项目 500 万 - 2000 万 license）。
- 9-12 个月出 POC。

#### KPI

- 2-3 家头部 CP 签约 POC，首年营收 1000-3000 万。

## §3. Top 3 最值得投入

### 🥇 第一名：P1 中文版 Generative Playable Feed

P1 是唯一一个**同时**满足"市场空白 + 技术已验证 + 窗口明确 + 中国特殊优势"四个条件的机会。

**为什么是第一**:

(1) **三个北美 / 欧洲对标已跑通 PMF**:Rezona DAU 40 万、LoopIt 40 万、YouTube Playables Builder 正在公测（PROGRESS §2.5）。这意味着"text prompt → playable → feed → remix"这条路径不再是假设，而是已验证。中文版不需要创造新范式，只需要工程复刻 + 本地化。

(2) **真正的空白象限**：玩家全景图 1.4 中，Pipeline-Creator × 中文区 的玩家数 ≈ 0。罗布乐思（Roblox 中国）已停运，Core / Rec Room / Crayta 无中文版，剪映只做视频不做交互。这是 2015 年"视频剪辑"心智之前的空窗期。

(3) **绕开了 Runtime AI 的 Token 真空带**：生成在创作时（一次性），运行时完全 0 Token，单用户日均 AI 成本可控制在 0.05 元以内，甚至豆包生态内可打到 0。这直接解决了本项目主命题的核心矛盾。

(4) **中国独有的豆包 Token 补贴 + 剪映 UGC 心智 + 双小游戏分发**：三个乘数效应叠加，同样的产品在中国能做到北美 10x 的 DAU 效率。

(5) **窗口期刚好 12-18 个月**：字节和腾讯大概率会下场，但他们的组织效率决定了至少要 9-12 个月才会出第一版。独立团队跑在前面，拿到早期种子创作者 + prompt 数据，能形成 moat。

**唯一的大风险**：内容审核成本。但这是可工程化的问题，不是技术不确定性。

### 🥈 第二名：P3 游戏壳 ≠ 游戏 陪伴 / 教育产品

P3 胜在**完全绕开版号 + 订阅 ARPU 3-5x**。星野 / 猫箱 已经用 5000 万 MAU 证明了陪伴赛道 PMF，但这些产品在"游戏核"上做得并不深（主要是对话，没有养成树 / 好感度升级 / 家族关系 / 任务系统）。一个 PM 如果能把"AI 对话 + 深度养成游戏核"做扎实，就能在红海里切出细分市场。教育英语对标多邻国 Max 也是同理。

风险是陪伴赛道头部效应已经很强，但订阅模型下哪怕只做 5 万付费用户，MRR 也有 150 万，活得很舒服。

### 🥉 第三名：P2 端侧小模型 AI-native 小游戏

P2 是技术含量最高、护城河最深的一条路，但时间窗更远（18-24 个月）。核心赌注是"端侧 3B 模型 + WebGPU 在 2026 年变成主流"。如果赌对，整个 Runtime AI × IAA 小游戏赛道会被重新打开（Token 成本 = 0 改变经济模型）。现在切进去是早期玩家优势。

缺点是技术难度大、兼容性痛、玩法设计还需要探索。适合有强技术背景的团队，不适合 PM 独立创业。

## §4. 给不同类型创业者的建议

### 个人独立开发者（1 人或夫妻档）

- **首推 P6 AI 视觉小说**:4-6 个月可出 MVP,Steam 付费分发不需要版号，80 万启动资金，单机品类，不需要做社区运营，质量上限决定成败。典型画像 — 有美术 / 文字功底的独立开发者。
- **次推 P5 Telegram Mini App**：如果有加密资产发行经验或海外渠道，Telegram + AI 小游戏是极好的单兵作战战场。

### 3-10 人小团队（有 200-500 万预算）

- **首推 P1 中文版 GPF**:5 人可以做起来，核心是快。豆包生态内 + 抖音小游戏 分发。
- **次推 P3 陪伴 / 教育**：若团队有 ToC 运营 + 订阅体系经验，陪伴 / 教育垂类的订阅模型回报最稳。
- **三推 P7 Playable Ads 工具**：若团队有 B 端销售资源 + 广告行业人脉，这条是最稳的现金流。

### 有融资的创业公司（A 轮 500-2000 万）

- **首推 P2 端侧 AI-native 小游戏**：资金充足才能扛住 6-9 个月的 R&D 空窗期。技术护城河最深。
- **次推 P4 AI NPC 中间件 SDK**:B 端 SaaS 打法需要长时间关系销售，融资是必要的。

### 中大型公司新业务线（字节 / 腾讯 / 网易 / 米哈游 内部）

- **首推 P1 的字节 / 腾讯 官方版本**：字节的优势是豆包 + 抖音小游戏 + 剪映用户；腾讯的优势是微信小游戏 + QQ 社交。官方做 GPF 是必然，时间问题。
- **次推 P8 Runtime AI for AAA**：网易伏羲 / 米哈游 内部已有基础，对外商业化 License 能做成新业务线。
