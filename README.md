# AI 小游戏与 Playable 体验赛道研究

> 2022-2026 · 创建 2026-04-13 · 进行中

---

有一个奇怪的事实：过去两年所有人都在谈"AI 生成游戏"，但真正把 LLM 塞进运行时的小游戏，几乎没有一家活过半年。不是做不出来，是做出来就亏钱。一局 2 分钟的消除游戏，玩家贡献的广告 eCPM 算下来不到一分钱，而背后一次 GPT-4o 调用的 token 成本可能就是三分。这就是我花了三天研究这个赛道之后，最想先告诉你的一句话：**Runtime AI × 小游戏是一块 token 经济学的真空带**，谁硬闯谁死。

真正有意思的事情发生在绕路上。2025 年下半年冒出来一个新物种，我暂且叫它 Generative Playable Feed（GPF）——把生成过程从"玩家开一局"挪到"开发者开一次"，再把产出物灌进类似短视频 feed 的消费流里。Rezona、LoopIt、YouTube Playables Builder 是三个样貌完全不同、但踩在同一条缝上的代表。它们共同回答了一个问题：当你不能让每个玩家都烧 token 的时候，怎么让 AI 还能成为护城河。

这份研究覆盖 7 个模块、92 家玩家的全景图，面向四类读者分别交付。如果让我把结论压到一张便签上，它是这样的：中国市场的 Pipeline-Creator 象限目前零玩家，窗口期 12-24 个月，之后海外巨头本地化或字节系内部团队进场，缝就焊死了。

## 几个需要先说清楚的约定

研究语言是中文，品牌名和技术术语保留原文——"Playable"翻译成"可玩广告"会丢掉一半语义，硬译不如不译。

深度是 L2 研究级，意思是对每一家公司我们要追到产品细节、团队背景和最近一轮融资条款，而不是停在"据报道"的层面。

地域分三层。Tier 1 是中国和美国，双中心并重，因为这两个市场的供给侧逻辑完全不同，缺一个就看不清整张图。Tier 2 是日本和韩国，Tier 3 是欧洲 indie 场景和东南亚。越往外层信息密度越低，但有几家关键公司就藏在 Tier 3 里，比如我们在 05 会提到的那家罗马尼亚工作室。

工具链上跑的是 Claude 和 Gemini 3.1 Pro 双路并行，一方卡壳另一方补位。这不是技术炫耀，是因为前两次研究里 Claude 在日韩语料上明显弱于 Gemini，而 Gemini 在中文长尾品牌识别上反过来弱于 Claude。互救是唯一能保住完成度的做法。

完整研究计划见 [research-plan.md](research-plan.md）。
