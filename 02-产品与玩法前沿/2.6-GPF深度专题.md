# 2.6 GPF 深度专题 — Generative Playable Feed 能不能成为 AI 游戏第一个正现金流品种

> 状态：DRAFT v0.1 · 2026-04-13 · 骨架先落，分节 Edit 填充

---

## Part 0 — 钩子

我花了两周追这个品类，结论先抛出来：GPF 可能是整个 AI 游戏赛道里目前唯一一个单位经济学有机会跑通的品种，但它跑通的前提条件，至少有三条现在还没被证明。

这句话需要拆一下才不显得夸张。整个 2023-2025 的 AI 游戏叙事都卡在一个算术问题上——GPT-4o-mini 一小时对话大约烧 $0.014，小游戏 IAA 的 ARPDAU 连 $0.05 都拉不满，AI 税在毛利里硬吃 15-40%[S-2.6-22]。Character.AI 28M MAU 的最终结局是被 Google 用 $2.7B 授权收编，Inflection 被微软 $650M reverse acqui-hire 吸走，2024 年 C 端独立 IPO 路径基本关闭[S-2.6-23]。所有 Runtime AI × 小游戏组合都死在同一个算术上。

GPF(Generative Playable Feed）的切入点恰恰是把 AI 从 runtime 挪到 pipeline——创作时烧一次 token，消费时零调用，靠 UGC 飞轮把单次生成成本摊薄到每次游玩 $0.0005 以下。这是目前观察到的唯一一个三难问题（Token 经济学 / 内容枯竭 / 冷启动）同时被触及的形态。但"触及"不等于"跑通"。三个未被证明的前提，我会在 Part 1.3 里一条条拆。先说结论：如果你只看到一半的故事而把 GPF 当作 AI 游戏的银弹，你大概率会低估它被平台化吃掉的速度。

## Part 1 — Facts

### 1.1 品类定义：Generative、Playable、Feed，三词缺一不可

GPF 这个名字是我在 2.5 节提议的，现在 2.6 节的工作是把它压实。品类的完整定义只需要一行：用户以自然语言 prompt 让 AI 生成 5-30 秒的 playable，作品进入竖屏无尽流被他人消费，任何观众都可以一键 remix 成衍生版本。三个词缺一都不成立——去掉 Generative 就退化成 Roblox，去掉 Playable 就退化成 TikTok，去掉 Feed 就退化成 Rosebud 式开发工具。

一个常被忽略的事实是 GPF 的成立窗口是 2024 下半年才真正打开的。三重技术同时到位缺一不可。第一是 LLM 代码生成在 2024 年 GPT-4o / Claude 3.5 Sonnet 这代模型上首次稳定到能产出可运行的 Phaser/PlayCanvas 脚本[S-2.6-18]；第二是 Diffusion 素材在 SDXL/Flux 迭代后可以以 $0.002-0.01 单次的成本产出风格一致的精灵与背景；第三是短视频竖屏消费姿势已经被 TikTok/Reels/Shorts 训练成全球默认习惯。这三件事在 2023 年都缺一角，所以 GPF 2023 年做不起来，2024 Q4 才突然冒出 Rezona / Sekai / YouTube Playables Builder 连续三个独立发布。

我一直想把 GPF 和四个相邻品类的边界写清楚，因为命名冲突把分析搞得一团乱。相邻四类分别是：Playable Ads（广告公司创作 · 目的是导流 · 没有 feed）、Roblox/Dreams UGC（开发者创作 · 高门槛 · 无 AI）、AI 陪伴如 Character.AI(runtime 烧 token · 没有可玩产物）、AI-native Steam 游戏如 Suck Up！（开发者创作 · runtime AI · 非 feed 分发）。GPF 恰好在这四个的正中央留下的空位里长出来。下面这张对比表是我允许自己放的第一张表，因为它压缩的信息密度值得占用版面。

| 维度 | Playable Ads | GPF | Roblox UGC | AI-native Steam | AI 陪伴 |
|---|---|---|---|---|---|
| 创作者 | 广告公司 | 普通用户 | 开发者 | 开发者 | 无 |
| AI 位置 | 无 | Pipeline（创作端） | 辅助工具 | Runtime | Runtime |
| 产物时长 | 15-30s | 5-30s | 无限 | 数小时 | 无限 |
| 消费形态 | 广告位 | 竖屏 feed | 游戏目录 | 独立客户端 | 对话窗口 |
| Token 经济学 | 不适用 | 创作一次性摊销 | 不适用 | 持续烧 | 持续烧 |
| 单次生成成本 | 广告制作 $500+ | $0.1-0.5（推测） | 0 | 不适用 | 不适用 |

### 1.2 产品深度拆解

研究期间我给自己定了一个纪律：每家产品必须覆盖七个字段——基本信息、核心循环、生成延迟、生成方式、remix 机制、UGC 比例、变现假设。七个字段里第三到第六个是大部分行业报道会跳过的，但它们才是决定 GPF 经济学的关键变量。下面八家深写，三家简写。

#### 1.2.1 Rezona(MWM.ai · 法国）

**基本信息**。MWM SAS，巴黎，2012 年成立，2020 年 3 月 B 轮 5000 万欧元 Blisce 领投，Idinvest / Bpifrance / Aglaé Ventures / Xavier Niel 跟投[S-2.6-01]。公司以音乐创作 app(edjing Mix / Beatmaker Go）起家，累计 400M+ 下载，2023-2024 转型 AI 创作工具，Rezona 是其 2025 年 Q4 进入 GPF 赛道的第一枪。iOS id6752310101,Google Play ai.rezona.app 2025 年 12 月上线[S-2.6-02]。

**核心循环**。文字 prompt → AI 生成 5-15 秒 meme 风格迷你游戏 → 发布竖屏 feed → 其他用户消费 → 一键 remix 继承原 prompt 可二次修改。Rezona 明确把自己定位成"让 meme 变成可玩"的工具，平台口号是"最离谱的 meme 游戏平台"，目标是把 TikTok 发 meme 的习惯平移到 playable。

**生成延迟**。我在官方页面和 App Store 描述中没找到官方数字，根据 LLM + Diffusion 合成 pipeline 的经验值推测为 10-40 秒区间，属于"可接受但不惊艳"的水平。这个数字如果能压到 5 秒以内，remix 飞轮的转速会直接翻倍。

**生成方式**。我的判断是模板 + 参数填充，不是 0 代码生成。证据有三：iOS 包体估计约 200MB 量级指向内置 runtime 和预置素材[S-2.6-02];meme 游戏的玩法类型有限（tap/swipe/physics/avoid），天然适合模板化；MWM 作为创作工具公司的技术路径一贯是"高可控 + 强模板"，不是 Websim 那种开放 HTML 生成。

**Remix 机制**。fork 式 + 参数调整混合。用户 remix 时继承原 prompt 但可编辑，类似 GitHub fork 的 UI 语义。这个设计对后端 provenance 要求较高——必须保留原 seed / 参数 / 素材 ID 才能做到可复现的局部修改。

**UGC 比例**。未披露，但"thousands of community-made mini-games"的官方措辞暗示 creator 池已达四位数[S-2.6-02]。如果 creator:consumer 比例低于 1:100，经济学不成立，见 1.3 节推导。

**变现假设**。当前无明显付费墙，推测走广告 + 未来订阅。MWM 旗下 edjing Mix 的订阅 ARPU 约 $3-4/月（推测，参照 App Store 榜单同类），如果 Rezona 能达到同量级变现而非依赖广告，单位经济学会好看很多。我的判断是 MWM 会用订阅托底，这也是它从音乐 app 带来的基因。

#### 1.2.2 LoopIt(MWM.ai · 法国）

**基本信息**。同 MWM 出品，iOS id6755859360,2025 年 Q4 仅 iOS 首发，比 Rezona 晚一个月左右[S-2.6-03]。定位是"Make Playables"，面向 ASMR / 互动 meme / 小玩具 / 互动艺术，不强调"游戏"二字。

**核心循环**。与 Rezona 同构但终点不同——LoopIt 的产物是"无目标互动玩具"(fidget / loop / ASMR），没有胜负或积分，用户的 engagement 是触感和审美而非挑战。这让它的 feed 消费时长更接近 ASMR 视频（30-90 秒）而非游戏（5-15 秒）。

**生成延迟 / 方式**。与 Rezona 共享同一套后端 AI pipeline（推测，基于 MWM 官网两款产品的 URL 结构和品牌相似度[S-2.6-03]），但前端 runtime 显然不同：LoopIt 强调粒子 / 物理 / 音效，Rezona 强调玩法机制。

**Remix 机制 / UGC 比例 / 变现**。与 Rezona 类似，此处不重复。我对 LoopIt 的判断是 MWM 的"品类 A/B 卡位"，在 GPF 子品类尚未定型前用双品牌占住"有目标迷你游戏"和"无目标互动玩具"两块地，留存数据出来后再把资源倾斜到赢的一边。这和 Meta 同时推 Instagram + Threads 的策略同构。

**一个我没想通的点**。如果 MWM 的战略是卡位，那为什么两款产品共享后端却没有统一品牌入口？我的猜测是它在赌"消费者场景切分"——想做 meme 游戏的人和想做 ASMR 玩具的人是不同 persona，统一品牌会稀释双方。但这个赌注要到 2026 Q2 留存数据出来才能验证。

#### 1.2.3 YouTube Playables Builder(Google · 美国）

**基本信息**。Google / YouTube 2025 年 12 月中旬开启闭测，用 Gemini 3 作为游戏生成底座，beta 目前限 US / Canada / UK / Australia，需要 active YouTube channel + 申请审批[S-2.6-04][S-2.6-05]。这是 YouTube Playables 产品线（2023 年上线的人工策划 HTML5 游戏库）的 AI 升级版。

**核心循环**。creator 在 YouTube Studio 内写文字 / 图片 / 视频 prompt → Gemini 3 转成可玩游戏逻辑 → 发布到频道或 Shorts → 订阅者在 YouTube 主 app 内直接游玩 → YouTube 算法分发。注意两个关键点：消费端不需要独立 app，直接跑在 YouTube 3B+ MAU 上；creator 已有受众，冷启动成本为零。

**生成延迟**。未披露，但 Gemini 3 的 token 速度比 GPT-4o 快 30-50%(Google 官方 benchmark，推测），理论上可压到 15-30 秒。

**生成方式**。基于 Gemini 3 + YouTube 内置 runtime，大概率是模板 + 参数的混合模式。Google 的战略不是"让创作者自由生成一切"，而是"让创作者在 YouTube 定义的游戏框架内生成 variations"。

**Remix 机制**。目前闭测版未明确是否支持 remix，我的判断是 2026 Q2 正式上线时会加入，因为不加 remix 就无法形成飞轮，YouTube 产品团队不会不懂这一点。

**UGC 比例 / 变现**。Google 的变现模型是 YouTube 现成的广告 + Premium 分成。这是 GPF 赛道里唯一一家不用自己发明变现路径的玩家。**也是为什么我认为 YouTube Playables Builder 对独立 GPF 创业公司构成近乎致命的威胁。**

**我的判断**。YouTube Playables Builder 会在 2026 年内完成正式发布，12-24 个月内会把独立 app 的生存空间压缩到 YouTube 不屑于做的边缘细分（meme / ASMR / adult 等）。Rezona / LoopIt 的 exit 路径很可能不是 IPO 而是被 TikTok 或 Snap 收购作为防御性并购，估值区间 $50-200M。

#### 1.2.4 Oasis / Decart AI（以色列 · 以色列 + 美国）

**基本信息**。Decart AI，以色列，2023 年由 Eliya Leitersdorf 和 Moshe Shalev 创立，2025 年 8 月 B 轮 1 亿美元，估值 31 亿美元，总融资达 1.53 亿美元，Sequoia / Benchmark / Zeev / Aleph 出资[S-2.6-06][S-2.6-07]。2024 年 10 月 31 日发布 Oasis,2025 年后期推出 Oasis 2.0,1080p 30fps[S-2.6-08][S-2.6-09]。

**核心循环**。Oasis 是一个用 next-frame prediction 训练的世界模型，吃键盘鼠标输入，吐下一帧画面——整个 Minecraft-like 体验没有引擎，完全靠模型推理[S-2.6-10]。严格说 Oasis 还不是 GPF（它没有 feed，也没有 creator remix 机制），但它是 GPF 的技术极限形态：把"生成 playable"的概念推到"每一帧都是生成的"。

**生成延迟**。这是 Oasis 最反常识的数据点——每 0.04 秒生成一帧，比 SOTA 模型快 100 倍[S-2.6-10]。这背后是 Decart 和 Etched 合作的专用芯片加速。为什么提这个数字重要？因为它暗示 Runtime AI 的成本曲线可能比我两周前以为的更陡峭——一旦专用硬件规模化，Runtime GPF 也许不是死路。

**生成方式 / Remix / 变现**。Oasis 的意义不在当下变现（上线三天破百万用户但免费[S-2.6-07]），而在它重新定义了"playable 的生成粒度"。如果下一代 Oasis 3.0 支持多人 + 自定义世界 + 分享链接，它就从"技术 demo"变成真正的 GPF 玩家。

**我的判断**。Decart 的 $3.1B 估值赌的不是 Oasis 本身能变现，而是"real-time generative interaction"这个基础能力会变成下一代消费 AI 的底座。如果赌赢，它是 GPF 赛道的 NVIDIA；如果赌输，它会变成一个被收购的牛 lab。

#### 1.2.5 Mirage 2 / Dynamics Lab（美国 + 以色列）

**基本信息**。Dynamics Lab，小于 10 人团队，Innovation Authority / Labs02 / OurCrowd / Takwin 投资，具体金额未披露[S-2.6-11]。2025 年 8 月 Gamescom 发布 Mirage 2，定位是"世界上第一个 AI-native 实时 UGC 游戏引擎"[S-2.6-12]。

**核心循环**。用户上传草图或照片 → AI 转为可交互 3D 世界 → 玩家在世界里用文字命令实时修改游戏[S-2.6-13]。核心差异是"实时修改"——不是生成完了就定格，而是玩的过程中可以 prompt 改世界。

**生成延迟**。未披露具体秒数，但官方强调"real time"，推测从图片输入到可玩世界 30-90 秒区间。

**生成方式**。世界模型路线，和 Oasis 同源技术但工具化封装更重。Mirage 更接近"让普通人做 Minecraft",Oasis 更接近"模型替代引擎"。

**Remix 机制**。天然支持——因为"玩的时候改世界"本身就是 remix 的一种形态。这让 Mirage 在技术上跳过了 fork 式 remix 的 UX 复杂度。

**变现 / 商业化**。2025 年 10 月免费层开放，未见付费墙[S-2.6-11]。小团队 + 免费层 + 无融资披露，说明还在产品验证阶段，12 个月内大概率有 Series A。

**我的判断**。Mirage 2 技术路线最激进也最有故事，但它和 Decart 一样面临同一个问题——世界模型的单次推理成本远高于 Pipeline 模板路线，如果不能靠专用硬件或端侧模型压成本，商业化窗口会比 Rezona 晚 2-3 年。

#### 1.2.6 Sekai: Create · Remix · Play（美国）

**基本信息**。Sekai 总部 Torrance CA,2024 年成立，$3.1M 种子，Azuki Labs / Hashed / Orange DAO / Zypsy / a16z 出资[S-2.6-14][S-2.6-15]。iOS id6504506653,Google Play chat.sekai.app[S-2.6-16]。注意：这家 Sekai 不是日本的 Sekai Lab 众包工厂，是完全不同的公司，名字撞车导致大量报道混淆。

**核心循环**。Sekai 自称"TikTok for mini-apps"，用户描述想要的东西 → AI 生成可玩 mini app → 无尽 feed 浏览 → 一键 remix → 分享链接到群聊让朋友一起玩[S-2.6-17]。产物类型比 Rezona 更宽——包括迷你游戏、约会决策器、投票轮盘、粉丝互动卡片等。

**生成延迟 / 方式**。官方口径"seconds"，推测 5-20 秒，快于 Rezona。生成方式也是模板 + 参数，但模板池比 Rezona 更杂。

**Remix 机制**。明确的 remix 按钮，可改 theme / 规则 / 风格。Sekai 的 remix UX 是我见过 GPF 产品里最直观的。

**UGC 比例 / 变现**。未披露。$3.1M 种子意味着至少 18-24 个月的 runway，变现压力不大，当前阶段重点是验证 remix 飞轮。

**我一直没想通的点**。Sekai 把 "mini app" 而不是 "mini game" 当作产物定位，这是一个有意的品类扩展还是品类模糊？如果是扩展，它赌的是 GPF 会演化成"personal software feed"——每个人发布的不是游戏而是"一个能用的小工具"；如果是模糊，那就是创始人对 GPF 品类本身的定义还没想清楚。我倾向前者，但没有证据。

#### 1.2.7 Whispers from the Star / Anuttacon(miHoYo 系 · 美国）

**基本信息**。Anuttacon 由 miHoYo 创始人蔡浩宇创立，2025 年 3 月公开 Whispers from the Star 并启动 iOS 闭测，仅限 iPhone 12 及以上[S-2.6-19][S-2.6-20]。严格说这不是纯 GPF——它没有 feed 也没有 creator remix，我把它放进来是因为它代表 GPF 的"AAA-native 相邻形态"。

**核心循环**。玩家和 AI 角色 Stella 实时对话（支持文本 / 语音 / 视频），对话决定剧情走向和角色命运[S-2.6-21]。Stella 的反应 / 情绪 / 行动都是 AI 实时生成，画面 100% 实时渲染。

**生成延迟 / 方式**。实时对话需要延迟 < 2 秒，这是典型 Runtime AI 架构，和 GPF 的 Pipeline AI 完全相反。**这家公司值得进入 GPF 报告的理由不是它是 GPF，而是它代表"蔡浩宇级别的 AAA 创始人都没选 GPF 路线"——这件事本身是一个强信号**。AAA 级团队看到的 AI 游戏未来是 Runtime + 情感关系，不是 Pipeline + UGC 飞轮。

**变现 / 商业化**。未披露，但参考 miHoYo 系游戏的抽卡 + 订阅常规路径，Whispers 可能会走月订阅 $10-20 的情感陪伴模型。

**我的判断**。Whispers 和 Rezona 代表 AI 游戏的两条可能路径——一条走深度少量用户 LTV，一条走浅度大量用户 UGC。两条都可能对，但我认为 GPF 路线的市场天花板更大而 Whispers 路线的单位 ARPU 更高。如果我是 LP，我会两边各下一注。

#### 1.2.8 Rosebud AI / Websim / Upit / Replit Game Builder（简写）

这四家严格说不在 GPF 品类核心里，但它们共同组成了"GPF 的上游技术祖先"池，不列出来会让整个地图失真。

**Rosebud AI**：旧金山，text-to-game 先驱（2023 年就开始），2024 年 Series A 报道 $15-25M（数据源冲突，以 Crunchbase 为准[S-2.6-24]），定位是"Vibe Coding"+ 资产管线集成，产物是 3D/2D 游戏项目而非 feed playable。Rosebud 没有 feed 也没有 remix 飞轮，它更像"给创作者的 AI Game IDE"，是 Pipeline-Dev 象限而不是 Pipeline-Creator 象限。

**Websim**:2024 年 4 月上线，Anthropic 系，底座是 Claude 3.5 Sonnet[S-2.6-25]。产物不只是游戏，还包括假网页、模拟器、互动叙事。Websim 代表"完全开放 HTML 生成"的极端路线，优点是自由度极高，缺点是稳定性和成本控制不及格——我的判断是 Websim 永远跑不通 GPF 经济学，但会作为"奇观展示场"存在。

**Upit**:Web 端，面向 GenZ creator，文字/图片 prompt 生成 3D/2D mini game，融资不详推测早期种子。Upit 最接近 GPF 的形态，但 Web-first 让它的 retention 结构和 mobile-first 的 Rezona 完全不同。

**Replit AI Game Builder**:Replit 2024 年 Agent 发布后强化的游戏生成入口，产物是源代码 + 可部署 web game，目标受众是开发者和 teen coder。Replit 不是 GPF，是 Pipeline-Dev 的代表，但它让"14 岁小孩用 prompt 做游戏"这件事第一次变得可重复。

#### 1.2.9 中国候选：豆包游戏 / 抖音小游戏 × 即梦 / 网易

中国 GPF 空白的大小，我在研究两周后还是有点震惊。2.5 节的 S-2.5-13 已经说过字节 / 快手 / 微信 / B 站都没有公开的 GPF 产品，2.6 节再查一轮证实没变。

**豆包游戏**。字节豆包 2025 年 3 月起升级 AI 编程能力，官方口径"3 分钟做出一款小游戏"[S-2.6-26]，能生成 HTML + JS 小游戏如贪吃蛇、消消乐。**但关键差别是**：豆包游戏的产物是"个人生成，自己玩 / 分享链接给朋友"，没有竖屏 feed，没有 remix 机制，没有 creator → consumer 杠杆。它是 Rosebud 中国版而不是 Rezona 中国版。

**抖音小游戏 × 即梦**。抖音小游戏月活 3 亿以上（2024 年字节官方），即梦 2024 年上线主要做图 / 视频生成。两者至今没有合并接口。我的判断是字节 2026 年内会合并"豆包 AI 生成 + 抖音小游戏分发 + 即梦素材"三条线，形成真正意义的中国 GPF 入口——但这个动作现在还没发生，对独立创业团队来说窗口就是这个未发生期。

**网易 / 腾讯 / 快手**。网易在 AIGC 方向的动作在逆水寒端游的 AI NPC(Runtime AI），和 GPF 完全不是一条路。腾讯微信小游戏是分发渠道，历史上对 UGC 创作器的态度保守，不会亲自做 Rezona。快手可灵专注视频生成，游戏线零动作。

**我的判断**。中国 GPF 的窗口期是 2026 全年，2027 Q1 前字节大概率会关门——不是因为字节一定能做成，而是因为一旦字节把豆包游戏 + 抖音 feed + 即梦素材三件套合并，独立创业团队的渠道护城河就被直接拆掉。

### 1.3 经济模型实测

开场抛的三条未被证明的前提，这一节要拆开写。但先从最基础的会计位置移动讲起——这是 GPF 与所有 Runtime AI 形态的本质差别。

**Pipeline AI 与 Runtime AI 的会计位置**。Runtime AI 里 token 消耗是经营性支出（COGS），每个玩家每小时都在烧；Pipeline AI 里 token 消耗是一次性资本化支出（CapEx-like），创作时烧一次，消费时为零。这不是省钱，是换了一个科目。两个科目的要命差别在于可摊销性——CapEx 可以随消费量线性稀释，而 COGS 是硬成本，玩家越多亏得越多。Character.AI 每新增一个 DAU 就新增 token 烧率，GPF 每新增一个 DAU 只新增 CDN 和广告请求。

**单次生成成本测算**。我按公开 API 价格做了个最小可行分解：一次 GPT-4o 级别代码生成，如果 prompt 压到 500-1000 tokens、产出 2000-4000 tokens，单次成本 $0.01-0.03;Diffusion 素材生成按 SDXL Turbo / Flux Schnell 的价格是 $0.002-0.01/张，一个游戏按 4-10 张素材算 $0.008-0.10；加上若干辅助调用（风格一致性 check、代码 lint、音效生成），单次生成总成本 $0.05-0.20 是合理区间[S-2.6-18]。如果走自建推理或 prompt 缓存，可以再压 30-50%。MWM / Sekai 的真实成本我没有证据，但 $0.1 以下是大概率的。

**Creator:Consumer 临界值推导**。这里有一个简单的收支方程。设单次生成成本 $0.15，单次 play 的变现（广告 $5 eCPM 下）约 $0.005，要让经济学成立必须满足"平均每个生成作品的总 play 数 × 单次 play 变现 > 生成成本"。代入得到临界 play 数 = 0.15 / 0.005 = 30 次。看起来很低，但这是**平均值**——正态分布下的长尾使得大部分生成作品只被玩 1-3 次，少数头部作品被玩几千次。要让平均数达到 30,feed 推荐算法必须把头部作品的 play 比例推到总 play 的 70%+。这就是为什么 GPF 没有好的算法就必死。

**Creator:Consumer 比例的几何约束**。如果平均每个 creator 一天生成 1 个作品，平均每个 consumer 一天 play 10 个作品，那么 creator:consumer = 1:10 就能达到供需平衡。但问题是新作品的边际效用递减——feed 需要持续喂新内容，实际需要的是 1:100 左右的比例。TikTok 视频 feed 的 creator:consumer 大约是 1:20（推测，基于公开报道），说明 GPF 的门槛还要更低才能跑通。Rezona 的 prompt-only 输入把创作门槛压到了 TikTok 量级，这是正确的方向，但还没被数据证实[S-2.6-02]。

**摊薄目标**。要让单次 play 的 AI 成本 < $0.0005，需要每个生成作品被 play > 300 次。根据 TikTok 视频作品的 play 分布（top 10% 占 70% 播放，推测），如果 GPF 的分布类似，那只有头部 10% 能过 300 次门槛，其余 90% 亏本。这个模式要成立，**必须靠头部养长尾**——平台经济学的老问题，和 YouTube / TikTok 的逻辑完全一致。

**eCPM 天花板问题**。这是三个未被证明前提中最棘手的一条。小游戏传统广告模式是每 3-5 分钟一次插屏，Rezona 的 5-15 秒游戏根本装不下传统插屏。候选方案有三条：生成前广告（prompt 输入后看 5 秒广告才开始生成）、remix 解锁广告（remix 前看广告）、feed 流内原生广告位（每 10 个游戏插 1 条广告）。这三条路都没有成熟数据。eCPM 下限推测 $1-3（比传统小游戏 $5-15 低一个数量级），上限要靠"生成即广告位"的新变现位发明。**这是 GPF 经济学的第一个未被证明前提**。

**补贴期时长推算**。MWM 2020 年 5000 万欧融资距今约 6 年，假设剩余现金 $10-20M 量级（推测，无披露），Rezona + LoopIt 两款产品的云推理 + 人员 + 补贴合计烧率推测 $2-3M/月，纯靠现有融资能撑 8-12 个月。如果 2026 Q3 前没有新一轮融资或明显留存证据，MWM 会被迫收缩战线或被收购。**这是第二个未被证明前提**:GPF 团队能否在融资窗口内跑出留存曲线。

**留存曲线假设**。传统小游戏 D1 30-40%、D7 10-15%、D30 3-5%。GPF 因为 feed 消费模式更像 TikTok,D1 可能高到 50-60%（看完就走但隔日回来容易），但 D30 可能反而更低（新鲜感衰减快）。这只是假设，所有产品都没公开数据。**这是第三个未被证明前提**:GPF 的留存曲线到底像 TikTok 还是像传统小游戏。

**和 Character.AI / AI Dungeon 的对比**。Character.AI 28M MAU 烧 $50M/年推理（推测，基于公开算力估算），ARPU 不到 $1，最终被 Google $2.7B 授权收编。AI Dungeon 10 万付费订阅月烧 token 成本占 ARPU 30-40%(Latitude 公开过早期数据）。GPF 如果做成同等 MAU，理论上推理成本能降到 Character.AI 的 5-10%（因为 Pipeline 不按会话烧）。这是 GPF 叙事最吸引投资人的那一页 PPT，但它的前提是上面三条未被证明的条件都成立。

**大厂距离评估**。下面六家对 GPF 的实际距离和可能形态：

**腾讯**。距离中等偏远。微信小游戏月活 5 亿是世界级分发渠道，但腾讯历史上对 UGC 创作器态度保守（微信没有 Effect House 等价物）。腾讯不会亲自做 GPF，但会通过投资 / 接入第三方的方式间接进场。最快进场时间 2026 Q4，最可能形态是微信小游戏平台给 GPF 开专用通道。

**Google**。已经在场。YouTube Playables Builder 2025 年 12 月 beta[S-2.6-05],2026 年内大概率正式发布。Google 的核心优势是 Gemini 3 + YouTube 分发 + 广告变现闭环，它是 GPF 赛道里唯一一家不需要解决任何经济学问题的玩家。进场形态已经确定——YouTube 内嵌 creator 入口。

**Character.AI**。距离极远且反向。Character.AI 是 Runtime AI 阵营的旗舰，被 Google 授权收编后方向转向"更轻的对话体验"，和 GPF 的 Pipeline 路线是两个宇宙。它进场 GPF 的概率低于 10%，更可能是被 GPF 品类成功反推压力而不是主动参与。

**Netflix**。距离远但意外地可能。Netflix Games 2023-2025 一直在探索互动娱乐，内部对 GPF 式的"短时可玩 + 社交分发"有兴趣，但 Netflix 的 DNA 是精品而非 UGC，强行下场会和公司文化冲突。我判断它更可能通过收购一家 GPF 创业公司（如 Sekai 或 Rezona）而不是自研。时间窗口 2026 下半年到 2027。

**Meta**。距离远且混乱。Meta 有 Reels（分发）、Llama 3.3（模型）、Horizon Worlds(UGC 平台）三块拼图但从未整合。Meta 做过的 UGC 游戏平台 Crayta 2024 年解散[S-2.5-19]，内部对 GPF 赛道缺乏明确主理人。我的判断是 Meta 不会主动下场，但会在 2027 年以防御性收购进场。

## Part 2 — Summary

**GPF 是不是 Token 经济学真空带的真正解**。我的答案是"目前是，但尚未被证明"。GPF 用会计位置移动绕过了 Runtime AI 的 COGS 陷阱，这在逻辑上是干净的，但绕过不等于解决。三个未被证明的前提——eCPM 能否撑住短时长、留存曲线能否像 TikTok 而非传统小游戏、creator:consumer 能否稳定超 1:100——任何一条崩掉都会让 GPF 退化成另一个"逻辑漂亮但数据难看"的品类。所以我对 GPF 的态度是有限看好：值得下注，但不应该当成银弹来推销。

**哪一类团队最有可能做成**。不是纯 AI 团队，也不是传统游戏老兵，而是"有创作工具基因的 AI 团队"。MWM 是最好的证据——它是音乐 / 视觉创作工具公司转型，不是游戏公司，所以它的产品直觉是"降低创作门槛 + 放大消费共振"，不是"设计机制 + 平衡数值"。游戏老兵做 GPF 会天然陷入"玩法深度陷阱"，想把 15 秒 meme 游戏做成 Mario——那不是 GPF，是别的东西。短视频平台（TikTok / 抖音）是终极优势玩家但惰性大，真正的创业窗口留给的是像 MWM / Sekai 这样的"小而快创作工具公司"。

**独立 app 和平台插件谁会赢**。短期独立 app 先跑出品类定义（因为独立团队敢做极端形态），长期平台插件收割大部分价值（因为分发和变现的垄断力不可抗）。这个模式和 Instagram vs Snapchat Stories 完全同构——Snapchat 发明 Stories,Instagram 抄去吃掉 80% 价值，Snapchat 靠剩下 20% 也活得下去。GPF 的对应关系是 Rezona / Sekai 发明品类，YouTube / TikTok 收割价值，独立 app 的天花板是 "被收购" 或 "小而美留在细分"。窗口期 12-24 个月，过了就关门。

**中国 GPF 的特殊性**。中国市场的 GPF 空白不是偶然，是 UGC + 生成式 AI 双重监管（网信办 UGC 审核 + 算法备案）导致的结构性滞后。字节最有能力破局但会被监管拖慢节奏，独立创业公司几乎无法独立做 GPF app（合规成本过高），真正的中国 GPF 只能以"平台内置工具"形式出现。这意味着中国版 Rezona 不会是一家独立公司，而会是"抖音 + 豆包"的某个 feature。对中国创业者的启示是：做 enabler 不做 app。

**Anuttacon 反信号**。miHoYo 系出身的蔡浩宇没选 GPF 路线，这是个需要认真对待的反信号。Whispers from the Star 赌的是"实时对话 + 情感 + 单用户深度 LTV"，和 GPF 的"浅度 + UGC + 飞轮"是两个正交维度[S-2.6-21]。两种路线都可能对，但说明最强大的 AAA 创始人更相信 Runtime 路线能做出高 ARPU 产品。这不是否定 GPF，而是提醒 GPF 叙事者：AI 游戏的未来可能是 GPF 和 Runtime 双轨，不是单轨替代。

**最容易被忽略的死法**。不是经济学、不是平台吃掉，而是监管。Rezona 的 meme 游戏强调名人脸 swap 和热点 parody,EU AI Act 2026 生效 + 美国州级深伪立法 + 中国算法备案，任何一条都可能突然提高合规成本。GPF 品类的法律脆弱性被严重低估，我判断 2026 下半年会出现第一个 GPF 公司因集体诉讼或监管处罚进入危机，这会成为品类最大的外部变量。

## Part 3 — Insights

### A. 产品 PM / 创业者视角

**A1 · 要不要下场**。下场，但只在接下来 12 个月内下场。窗口期 = 从现在到 YouTube Playables Builder 正式发布 + 字节豆包游戏 feed 化合并，保守估计 2027 Q1 关门。**里程碑**:Q3 2026 前做出内部 MVP(prompt → playable 端到端 20 秒内完成 + 10 个模板），Q4 前 10K 种子用户，2027 Q1 前 creator:consumer 数据达 1:50。**失败模式**：做成"给开发者的 AI 游戏工具"而不是"给 meme creator 的玩具"，这个定位偏移会让你变成下一个 Rosebud 而不是下一个 Rezona。

**A2 · What NOT to build**。不要做开放式生成（Websim 路线在稳定性和成本上至今不合格[S-2.6-25]）；不要做纯 Runtime AI 游戏（被 Token 经济学吃干）；不要做 3D / 重度玩法（延迟和成本都压不下来）；不要做 Web-first(mobile-first 的 retention 结构优势不可替代）；不要在开场就做中国版（先做跨境海外版再考虑合规回流，中国本土直接起做独立 app 的监管成本过高）。**失败模式**：任何偏离"5-30 秒 · 模板化 · 移动端 feed · meme 导向"的路线在 12 个月内都会因为烧不起钱或冷启动不了而死。

**A3 · 团队最小配置**。5-8 人：1 个有创作工具 PM 背景的 founder（不是游戏 PM）、1-2 个全栈 AI infra 工程师（懂 LLM prompt 缓存 + Diffusion pipeline）、1 个移动端 runtime 工程师（Phaser / PlayCanvas）、1 个算法工程师（feed 推荐）、1 个内容运营（冷启动时每天发 10 条作品撑 feed）、1 个 growth。**冷启动打法**：不要等 UGC 飞轮自然转动，前 3 个月用运营团队自己当 creator 把 feed 填到 200 个作品作为基线，之后靠种子用户激活飞轮。**失败模式**：相信"好产品自然会有 UGC"，冷启动前 90 天内 feed 长期少于 50 个作品就是死亡曲线。

### B. 投资人视角

**B1 · 估值区间与退出路径**。种子轮 $3-8M post（参考 Sekai $3.1M[S-2.6-15]),A 轮 $20-40M post,B 轮 $80-200M post（参考 Decart B 轮 $100M/$3.1B valuation 是上限异常值[S-2.6-06]）。退出路径清晰：2027-2028 年被 TikTok / Snap / Netflix 防御性收购，退出估值区间 $100-500M。**里程碑**：种子轮看 creator:consumer 达 1:30 + 月环比 50%,A 轮看 100K DAU + D30 10%,B 轮看 ARPU 证据或 eCPM 验证。**失败模式**：把 GPF 当作 Character.AI 式"高 MAU 低 ARPU" 故事投——GPF 的退出上限是千万级 ARR 的 creator tool，不是几亿 DAU 的消费社交，混淆这两个会导致高估值后无法兑现。

**B2 · 按阶段下注建议**。种子阶段（2026 上半年）——下 3-5 家赌 thesis 对，每家 $1-3M，接受 80% 失败率；A 轮阶段（2026 下半年）——聚焦留存数据最好的 1-2 家加注 $5-10M;B 轮阶段（2027）——避免跟 Sequoia / Benchmark 抢头部，转做二级市场 / secondary。**警示信号**：任何一家 GPF 公司在 A 轮时还没公开 creator:consumer 和 D30 数据的，pass；任何一家把估值锚定在 "Character.AI 类比"的，pass；任何一家 3 人以下团队自称要做"AI 一切"的，pass。

**B3 · 什么时候必须止损**。三个触发器任意一条触发就清仓：一、YouTube Playables Builder 正式发布且在 6 个月内 beta → GA 成功，意味着独立 app 的渠道护城河被拆；二、字节在中国正式推出"抖音 AI 游戏 feed"入口且月活破 1000 万，意味着中国通道被吃；三、行业内任一 GPF 公司因深伪 / IP / 监管被集体诉讼或政府处罚超过 $10M 量级，意味着品类法律风险具象化。**失败模式**：死守"再等半年"信仰而不止损，这是 2021 年 Web3 游戏投资人的同一错误。

### C. 行业观察视角

**C1 · GPF 对小游戏平台的冲击**。传统小游戏平台（微信小游戏、抖音小游戏、YouTube Playables 旧版）的内容生产模式是"专业开发者 + 策划审核 + 广告分发"。GPF 把生产端的专业性拆掉，内容产出速度提高 100 倍但质量方差变大 10 倍。**时间窗口**:2026 全年小游戏平台会被迫做出反应，要么开放 AI 生成通道（YouTube 模式）要么坚持策展路线（微信模式）。**里程碑**:2026 Q4 前至少一家头部小游戏平台会有 GPF 类入口或接入。**失败模式**：平台方误以为 GPF 是"低质量垃圾"，错过窗口期，2027 年被迫高价收购或自建追赶。

**C2 · GPF 会杀死哪些公司**。第一批死的是"AI + 游戏 + 独立 IDE"方向的早期创业公司（Rosebud / Ludo.ai / Jabali 等），它们没有 feed 就没有护城河；第二批是传统 HTML5 游戏工作室中 5 人以下、以 meme / casual 为主的团队，GPF 会把他们的产出成本归零；第三批是"Playable Ads 外包工作室",GPF 让广告主可以自生成 playable ads，外包需求下降 50%+。**时间窗口**:2026 下半年开始，2027 全年是洗牌期。**失败模式**：被杀的公司大多会尝试"加 AI 救自己"，但他们的 DNA 不对——真正的幸存策略是转型 enabler 或被收购。

**C3 · GPF 对 UGC 创作工具行业的影响**。GPF 把 creator economy 的定义从"内容创作 （视频 / 图 / 文）"扩到"可交互资产 (playable / tool / mini-app)"。这个扩展会反向压力 TikTok / Instagram / YouTube 重新定义 creator 的工具栈——不只是剪辑和特效，而是代码 + 模板 + 互动。**时间窗口**:2026-2028 三年内 creator tool 行业会出现一批"AI 小开发工具"新公司，其中 1-2 家会涨到 Figma 级别。**失败模式**:Adobe / Canva 级玩家如果在 2026 年内不进场，2028 年会发现创作工具市场被新品类重新划分，错失就是永久错失。

### D. 战略决策者视角

**D1 · 为什么做 GPF 比做 Runtime AI 更紧迫**。Runtime AI 的经济学还没跑通（Character.AI 的退出就是证据[S-2.6-23]），要等模型成本下降 10 倍（推测 2027-2028），这个时间窗口大厂可以慢慢跟进。GPF 的窗口相反——技术条件现在就成熟，品类定义权在 12-18 个月内会被抢光，谁先推出成熟产品谁就锁住位置。**Runtime AI 是长期战，GPF 是闪电战**。大厂如果在 2026 年不做 GPF，就意味着把一整块 playable 创作分发位拱手让给 YouTube 或新兴创业公司，而这块位的长期价值可能是千万级 DAU。

**D2 · 字节跳动怎么做**。立即启动"豆包游戏 + 抖音小游戏 feed + 即梦素材"三件套合并项目，目标 2026 Q3 内部 alpha,2026 Q4 在抖音内灰度 100 万用户。**里程碑**:Q2 2026 前完成技术方案选型（模板 vs 世界模型），Q3 前 MVP,Q4 前用户测试。组织结构建议是"AI 平台 + 抖音游戏 + 即梦"联合项目组向抖音总裁直接汇报，不要走豆包独立线。**失败模式**：被豆包的"对话 AI 助手"定位拖住，错过 feed 一侧；或者被抖音小游戏平台的"监管合规"顾虑拖住，错过产品一侧。

**D3 · 腾讯 / 网易怎么做**。腾讯应该放弃亲自做 GPF 的念头（基因不对，UGC 工具历来是腾讯弱项），改为投资 + 战略合作 + 微信小游戏平台开放通道三条腿。具体动作是 2026 年内投 2-3 家海外 GPF 创业公司各 $5-15M，同时在微信小游戏平台开"AI 生成游戏"审核绿色通道。网易同理但更极端——网易应该彻底放弃 GPF 跟进，把资源集中在 Runtime AI × AAA 的差异化路线（逆水寒 AI NPC 是起点而非终点）。**失败模式**：腾讯硬做微信版 Rezona，会重蹈 Crayta 的覆辙[S-2.5-19]。

**D4 · Google / Meta 怎么做**。Google 已经在正确路径上（YouTube Playables Builder + Gemini 3[S-2.6-04][S-2.6-05]），现在的问题不是战略而是执行速度——beta 到 GA 必须在 2026 Q2 前完成，否则 Sekai / Rezona 会在窗口期内抢走品类定义权。Meta 的情况更困难，Reels + Llama + Horizon 三块拼图从未整合，建议直接收购一家已有 GPF 产品的公司（Sekai 是最佳目标，估值可控 $30-80M）而不是自建。**失败模式**:Meta 继续内部孤岛式尝试，2027 年发现品类已定型时才启动收购，价格被抬到 $300M+ 级别。

## Part 5 — Gaps

GPF 品类的数据黑洞比我研究过的任何一个 AI 子品类都大，这本身是一个值得写进报告的观察。这一节我把所有我没研究到的东西明确摊开，不假装知道。

**黑洞一：所有产品的 MAU / DAU / 留存数据全部未公开**。MWM 是私营公司无财报，Sekai 种子期不披露，YouTube Playables 藏在 YouTube 主 app 内没单独数据，Rezona / LoopIt App Store 榜单可作次优代理但噪音大。整个 1.3 节的经济学论证是建立在"如果这些数字达到 X 则经济学成立"的条件式上，而不是"这些数字已经是 X"的断言式上。这是 GPF 研究最大的一块不确定性。

**黑洞二：单次生成成本的真实数字**。$0.05-0.20 的估算来自公开 API 价格和常识假设，但真实成本取决于 prompt 缓存效率、模型蒸馏、自建推理集群等公开信息之外的变量。MWM 或 Sekai 的真实单位成本可能比我的估算低 30-50%（如果用了自建推理）或高 30%（如果未优化）。

**黑洞三：creator:consumer 比例的真实分布**。我在文中反复用 1:100 作为临界值，但这个数字来自 TikTok 视频的类比而非 GPF 真实数据。GPF 的 creator:consumer 可能因为"创作即玩"的模糊边界（remix 一次算 creator 还是 consumer）而无法用传统方式度量。这个度量方法本身都还没被业界定义。

**黑洞四：eCPM 与短时长广告位的匹配方案**。5-30 秒 playable 能不能支撑传统插屏 / banner / incentivized ads，目前完全无公开数据。第一家在这条路上做出数据的公司会直接决定品类变现上限。

**黑洞五：Rezona 与 LoopIt 的 MWM 内部资源分配**。双品牌战略只能外部观察，无法确认团队是联合还是独立、预算比例如何、内部 KPI 是否一致。如果两边内部打架，MWM 在 GPF 赛道的执行力会打折扣。

**黑洞六：中国侧的非公开动作**。字节 / 腾讯 / 网易的 GPF 相关内部项目可能已经存在但未公开。只能通过专利 / 招聘 / 代码仓库间接验证，本次研究没有深挖这些 indirect signals。中国 GPF 的真实状态可能比我描述的更活跃也更保守，两个方向都有可能。

**黑洞七：可能存在但我没找到的产品**。GPF 是一个 2024 下半年才成型的品类，全球独立创业潮大概率还有 3-5 家我没检索到的玩家，尤其在韩国 / 东南亚 / 印度三个市场。本研究对日韩覆盖薄弱，这是明确缺陷。

**黑洞八：监管影响的量化评估**。EU AI Act / 美国州级深伪立法 / 中国算法备案对 GPF 品类的实际合规成本，我只能定性判断"会提高"但给不出"提高多少"。第一个被处罚的 GPF 公司出现之前，这条数据链缺失。

**黑洞九：Apple 系统级动作的不可见性**。苹果对 App Store 动态代码生成的审核标准一直是灰色地带——Rezona / Sekai 这类 playable 生成器理论上游走在 4.7 条款（可执行代码下载）边缘。更深一层的风险是，如果 Apple Intelligence 在 2026-2027 内推出系统级的原生互动 feed（比如"Messages 里直接生成互动贴纸/玩具"），整个 GPF 品类的分发前提会被釜底抽薪。这是我整份报告最没把握的一条外部变量。

**黑洞十：超级创作者的激励闭环**。即便 GPF 把创作门槛压到最低，能稳定写出高质量 prompt 做出爆款玩法的"超级创作者"仍然是稀缺资源——TikTok 和 YouTube 已经证明 creator 头部效应极强。如果 GPF 平台不能在 2026-2027 建立成熟的创作者分成体系（内购 + 广告分润 + 订阅收入分配），这批超级创作者会被 YouTube Playables / 抖音 feed 的成熟分成体系直接挖走。目前没有一家独立 GPF 团队对外谈过分成方案，这是个大坑。
