# 3.1 现有技术栈能力 — AI 小游戏与 Playable 体验（2022–2026）

> 目的：清点 2026 年 4 月前后，在"小游戏 + playable 广告"这一约束很强的场景下，底层 AI 技术栈能力的真实边界。避免只看 demo，追问"端到端是否可跑通 + 单位经济是否为正"。

## 一、背景与取数口径

本节限定在"小游戏形态"——HTML5 / 微信小游戏 / TikTok playable ad / iOS 14.5 后归因受限环境。这意味着技术评估的前提约束是：

- **包体 ≤ 5 MB**（微信小游戏主包上限在 2025 年放宽到 30 MB，但播放器仍以 4–8 MB 为主）
- **首屏 ≤ 3 秒**（playable 广告行业基准，CTR 会随加载时长指数下降）
- **单用户日成本 ≤ CPI**（典型休闲 iOS CPI 1.5–3 美元，Android 0.3–0.8 美元）
- **无服务器强制缓存 / 审核合规**（微信/字节/Meta 对云端 AI 调用有额外审核）

所有能力评估都要回答：在这些约束下是否可用，而非"实验室能跑"。数据来源综合 Artificial Analysis（2025 LLM benchmarks）、Poe、Replicate、a16z "State of Generative AI"、GDC 2025 AI Summit、以及开发者社群（Reddit r/gamedev、HackerNews、知乎小游戏板块）。

## 二、核心发现

### 2.1 LLM 能力边界（2026-Q1 快照）

| 模型 | 上下文 | 首 token 延迟 | 每 1M in/out token 价 | 角色一致性 | 可用场景 |
|---|---|---|---|---|---|
| GPT-4o | 128K | 0.4–0.8 s | 2.5 / 10 美元 | 中等，长对话后约 8% 破人设 | 高端 AI NPC、带语音的陪伴 |
| GPT-4o-mini | 128K | 0.3 s | 0.15 / 0.6 美元 | 一般 | 免费休闲游戏的对话底层 |
| Claude 3.5 Sonnet | 200K | 0.6 s | 3 / 15 美元 | 最好，长对话人设保持率 > 93% | 叙事向、有世界观设定的游戏 |
| Claude 3.5 Haiku | 200K | 0.4 s | 0.8 / 4 美元 | 良好 | 中端 playable，预算敏感 |
| Gemini 2.5 Flash | 1M | 0.3 s | 0.075 / 0.3 美元 | 一般 | 低成本高并发、playable ad |
| Gemini 2.5 Pro | 2M | 1.1 s | 1.25 / 5 美元 | 中等 | 多人游戏世界状态维护 |
| Llama 3.1 70B（自托管） | 128K | 依 GPU 而定 | ~0.6 / 0.8 美元（A100 自托管） | 中等 | 出海合规、数据私有化 |
| DeepSeek V3 / R1 | 128K | 0.5 s | 0.27 / 1.1 美元 | 中等 | 国内厂商低成本方案 |

**四条能力上限**：

1. **上下文不等于记忆**：200K–2M 的上下文只是滑动窗口，真正的"角色记得三天前发生的事"需要外接 RAG + embedding store，工程成本≥上限。
2. **角色一致性是概率问题**：即便是 Claude 3.5 Sonnet，长对话仍会"破人设"——一旦玩家故意试探或使用脏话，模型可能跳回"我是一个 AI"。商业上必须配合 guardrails。
3. **首 token 延迟 = 玩家体验**：playable 广告场景 > 1.5 s 首 token 就会让 CTR 下降 15–25%。这把昂贵的大模型直接排除出 playable ad。
4. **上下文贵**：2M 上下文的 Gemini 单次调用虽然便宜，但重复喂同一个世界设定会让成本线性增长。Prompt caching（Anthropic / OpenAI 2024 发布）把重复内容价格打到 1/10，是目前唯一可行的成本兜底。

### 2.2 Diffusion 在游戏美术的能力

- **SDXL / SD3** 开源权重，512×512 单张约 1–3 秒，成本 < 0.001 美元（自托管）。缺点：手、文字、连续帧一致性差。
- **Flux.1** 质量接近 Midjourney，商业许可 Flux Pro 通过 API 调用约 0.05 美元/张。
- **Midjourney v6.1** 质量最高，但无官方 API、不能嵌入商业管线。
- **ControlNet + LoRA** 可以做到"同一角色 20 张立绘"一致率≈70%；关键是"角色表情包 + 同人设"场景。
- **游戏美术工业化价差**：一个传统 2D 立绘约 800–3000 元/张；AI 生成 + 美术修图 ≈ 80–200 元/张。节省 5–15 倍。

能力上限：**视频 / 动态立绘的时间一致性仍是硬伤**，Runway Gen-3、Sora、可灵、Luma 都无法生成"同一角色 10 秒连续表演"且可商用。这也是"AI 动态立绘"为什么没跑通的主因。

### 2.3 Voice / TTS 能力

- **ElevenLabs Flash v2.5**：75 ms 延迟，0.15 美元/千字符，支持 32 种语言、语音克隆。是 AI NPC 实时对话的工业标准。
- **OpenAI TTS-1 HD**：0.03 美元/千字符，质量高但延迟 400 ms+，不适合实时。
- **OpenAI Realtime API（gpt-4o-realtime）**：端到端语音，输入 100 美元/1M audio token、输出 200 美元/1M。成本偏高但延迟最低（平均 320 ms）。
- **腾讯云语音 / 火山引擎**：国内合规可用，价格更低（约 ElevenLabs 的 1/3），但音色选择和情感表达不如 ElevenLabs。
- **Sesame / Kyutai Moshi**：2025 年出现的开源端到端语音模型，延迟 < 200 ms，但角色音色只有有限选项。

能力上限：**中文情感语音 + 低延迟 + 低成本**三者不可兼得。国内小游戏做 AI NPC 只能在"延迟 800 ms + 质量一般"和"延迟 200 ms + 贵 5 倍"之间二选一。

### 2.4 Agent / RL 框架

- **AutoGen / LangGraph / CrewAI**：让多个 LLM 扮演不同角色（游戏里如多个 NPC）互相对话。但在游戏里更常用的是"单 NPC agent + 工具调用"。
- **Anthropic Computer Use / OpenAI Operator**：2024–2025 发布的 agent 能力，在游戏里尝试作"AI 队友代玩游戏"——失败率仍很高（GDC 2025 报告，《Dota 2》AI 队友的胜率提升有限、但玩家满意度提升明显）。
- **经典 RL**：Unity ML-Agents、OpenAI Five 等技术仍在用，但对小游戏成本过高。
- **Inworld AI SDK**：把"角色设定 + 记忆 + 情感状态机 + TTS"打包成 Unity / Unreal 插件，开发者集成约 3–5 人日。2024 年免费额度下调后成本较高。
- **rct AI Chaos Box / 心智引擎**：国内同类方案，已接入字节若干立项游戏。

### 2.5 渲染 & 小游戏前端

- **WebGPU** 2023 Q2 随 Chrome 113 正式上线，到 2025 年底覆盖率 ≈ 75%（iOS 17+ 和 Chrome/Edge/Android 都支持），性能接近原生 60%。
- **Three.js / Babylon.js / PlayCanvas** 是 playable 广告和 H5 游戏标准栈。PlayCanvas 提供在线编辑和一键发布。
- **Unity WebGL** 包体普遍 8–20 MB，超过 playable ad 上限；Unity 2023 LTS 后推出的 Web 模板将最小包体压到 5 MB 以下但仍偏大。
- **Cocos Creator 3.8**：国内微信小游戏的事实标准，支持 WebGPU；包体控制 2–4 MB 是常态。
- **PixiJS / Phaser**：纯 2D playable，最轻量、首屏 < 1 s。

### 2.6 Token 成本 & 延迟对经济模型的影响

典型"AI 陪伴类休闲游戏"玩家单日消耗估计：

- 每回合对话 ≈ 400 input + 150 output token
- 每日约 30 回合 → 12K input + 4.5K output token/人/日
- GPT-4o-mini ≈ 0.0018 + 0.0027 = 0.0045 美元/人/日
- Gemini 2.5 Flash ≈ 0.0009 + 0.00135 = 0.00225 美元/人/日
- GPT-4o ≈ 0.030 + 0.045 = 0.075 美元/人/日

对比休闲游戏 ARPDAU（每日每用户收入）仅 0.02–0.05 美元 —— GPT-4o 直接亏 1.5–3 倍，GPT-4o-mini 才刚打平。**这是 AI 小游戏最大的商业瓶颈**。

## 三、案例 / 数据点

1. **Suck Up!**（Proxima Enterprises, 2023 Q4）：全 LLM NPC 吸血鬼游戏，Steam $9.99 买断，Metacritic 78。2024 年中披露单用户 LLM 成本约 2.5 美元/月，买断价覆盖 2–3 个月使用。
2. **AI Dungeon**（Latitude, 2019–2025）：从 GPT-2 演化到自研 griffin 模型，月活约 100 万，付费率约 6%，年收入约 2000 万美元。这是目前为止单款 AI 游戏的天花板。
3. **Inworld Origins**（Inworld Demo, 2023）：展示多 AI NPC 互相对话 + 玩家介入的侦探玩法，demo 火了但未成品。
4. **Character.ai**（2022–2024）：高峰期 DAU 2000 万，但谷歌 2024 年 8 月以 25 亿美元"收购团队 + 授权技术"——表明"纯 AI 陪伴"可以有用户、但难以独立上市。
5. **字节 Dreamina / Jimeng**：2024 年开始接 AI 美术管线到内部游戏研发，据公开访谈美术产能提升 3–5 倍。

## 四、Insights（四视角）

**A. 产品视角**
真正可跑通的"AI+小游戏"堆栈是：Cocos Creator 2D 壳 + GPT-4o-mini / DeepSeek V3 对话 + ElevenLabs Flash 语音 + SDXL + LoRA 静态立绘。任何试图把"实时视频 / 高保真 3D / 长时记忆"一起塞进来的项目都会碰技术墙。

**B. 开发者视角**
2026 年的"AI 游戏工具链"仍然割裂：LLM / TTS / 美术 / 记忆系统都是独立 API。真正的 SDK 赢家是能把"一个角色的声音 + 视觉 + 人格 + 记忆"封装成一个对象的工具（Inworld 走对了方向，但定价过贵）。

**C. 投资人视角**
底层模型价格每 9 个月下降约 4 倍（Epoch AI 追踪），这意味着 2024 年不跑通的单位经济，在 2026 年可能自动跑通——但这不是护城河。投资应该看"谁在价格下降后第一个吃到红利"，而不是赌当下的成本优势。

**D. 大厂战略视角**
字节 / 腾讯 / 米哈游的真实算盘是自研模型 + 自家分发 + 自有 GPU 集群，把 AI 单用户成本压到外部 API 的 1/10。这让任何纯"调用 API 做产品"的创业者在 2 年内会被大厂模型 + 分发碾压。差异化只能来自"数据 / 品牌 / 玩法 / 审核灰区"。
